本科毕业论文问题检查与汇总系统

> 面向计算机科学与技术/软件工程专业的本科毕业论文自动化审查工具，基于大模型 Agent 多步骤自动执行，生成结构化问题清单与修改建议。

---

项目简介

本提示词系统用于自动化审查"XX系统的设计与实现"类本科毕业论文，通过 14 个串行检查步骤 覆盖论文从选题到格式的全维度质量评估，最终产出完整问题清单、逐章汇总表、横向一致性校验报告。

系统采用单 Agent 多步骤自动执行模式：用户上传论文后，Agent 自动创建任务列表并按序执行，无需人工干预，等待最终结果即可。

---

核心特性

- 8 大检查维度：选题、文献、综合知识与技能、写作水平、格式规范、研究成果、交叉验证、性能测试数据防造假
- 分层审查架构：每个步骤分"基础审查层"和"一致性验证层"，先收集证据再做判断
- 前向预警机制：前序步骤发现风险时向后序步骤传递预警，避免问题遗漏
- 调用链拆解防造假（V2 新增）：将性能测试数据与系统接口设计文档跨章节关联，通过理论延迟下界估算识别数据造假
- 文件持久化：每步结果写入独立文件，防止上下文丢失，支持逐步追溯
- 问题汇总拆分：最终汇总分 3 个子任务执行，防止输出被截断

---

适用对象

- 计算机科学与技术 / 软件工程专业本科毕业论文指导教师
- 论文评审专家（本科层次）
- 需要自查论文质量的学生

适用论文类型："[系统名称]的设计与实现"类工科论文，重心在应用研究价值、工程实践完整性、技术方案合理性。

使用方法

环境要求

- [Kimi Agent](https://kimi.ai/)（或其他支持长上下文和文件持久化的大模型 Agent）
- 上传论文文件（PDF 或 Word 格式）

操作步骤

1. 上传论文文件到 Kimi Agent 对话窗口
2. 粘贴 [`论文问题检查提示词V2.md`](论文问题检查提示词V2.md) 的提示词部分内容
3. Agent 自动创建 14 个任务并按序执行，你只需等待最终结果
4. 从 `/mnt/agents/output/paper_review/` 目录下载全部审查结果文件，最重要的输出文档为：step09_问题汇总.md 

注意事项

- 提示词内容较长，确保完整粘贴，不要截断
- 每个步骤完成后 Agent 会自动汇报关键结果和文件路径
- 全部 14 个任务执行完毕后会输出最终汇总文件路径

输出文件结构

所有中间产物保存在 `/mnt/agents/output/paper_review/` 目录：

```
paper_review/
├── step01_基础信息.md           # 论文事实数据库（12项基础信息）
├── step02_选题审查.md           # 选题审查记录与问题清单
├── step03_文献审查.md           # 文献审查记录与问题清单
├── step04A_要素映射.md          # 需求→设计→实现→测试映射表
├── step04B_闭环验证.md          # 闭环验证问题清单、断裂率
├── step05A_语言规范.md          # 语言问题清单、术语一致性
├── step05B_逻辑结构.md          # 逻辑与结构问题清单
├── step06_格式审查.md           # 格式问题清单
├── step07_研究成果.md           # 研究成果审查、性能基础检查
├── step08_交叉验证.md           # 7项横向一致性校验结果
├── step08_5_性能测试审查.md     # 性能测试数据防造假检查（V2新增）
└── step09_问题汇总.md           # 最终汇总文件（完整清单+逐章汇总）
```

最终交付物：`step09_问题汇总.md` 包含：
- 8 个维度的问题清单
- 完整问题与修改建议汇总表（按问题编号排序，不截断）
- 逐章问题汇总表
- 横向一致性校验结果
- 总体概览（问题总数、严重/一般问题数）

---
```
优先级定义

优先级	       含义	                     典型场景	
P0	       严重，必须修改	       摘要与正文矛盾、核心章节缺失	
P1	       重要，建议修改	       图表不对应、逻辑断裂、闭环断裂	
P2	       一般，建议改进	       术语不一致、引用失当、格式错误	
P3	       轻微，可选改进	       病句、口语化、标点混用	
```
